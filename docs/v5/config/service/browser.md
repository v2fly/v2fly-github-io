# 浏览器转发模块

service.browser

浏览器转发模块（Browser Forwarder）允许 V2Ray 提供一个本地 Web 界面，用于在浏览器中控制和管理代理连接。通过此模块，用户可以通过浏览器界面便捷地切换出站连接、查看连接状态等。(v5.0+)

## 配置

```json
{
  "listenAddr": "127.0.0.1",
  "listenPort": 8080
}
```

> `listenAddr`: string

浏览器转发页面的本地监听地址。

* 默认值：`"127.0.0.1"`
* 推荐使用 `127.0.0.1` 以确保只能从本机访问
* 如需从局域网访问，可以设置为 `"0.0.0.0"`，但需注意安全风险

> `listenPort`: number

浏览器转发页面的本地监听端口。

* 必须是一个有效的端口号（1-65535）
* 确保该端口未被其他程序占用
* 常用端口如 8080、8081、8888 等

## 使用示例

### 基本配置

```json
{
  "services": {
    "browser": {
      "listenAddr": "127.0.0.1",
      "listenPort": 8080
    }
  }
}
```

启动 V2Ray 后，访问 `http://127.0.0.1:8080` 即可打开浏览器控制界面。

### 局域网访问配置

如果需要从局域网内的其他设备访问控制界面：

```json
{
  "services": {
    "browser": {
      "listenAddr": "0.0.0.0",
      "listenPort": 8888
    }
  }
}
```

:::warning 安全提醒
使用 `0.0.0.0` 时，任何能访问该端口的设备都可以控制 V2Ray。请确保：
* 使用防火墙限制访问
* 只在可信网络中使用
* 考虑配合其他认证机制使用
:::

### 自定义端口配置

```json
{
  "services": {
    "browser": {
      "listenAddr": "127.0.0.1",
      "listenPort": 9090
    }
  }
}
```

## 功能特性

通过浏览器转发模块，你可以：

* **可视化管理**：通过 Web 界面查看和管理代理配置
* **实时切换**：动态切换不同的出站连接
* **状态监控**：查看当前连接状态和统计信息
* **便捷操作**：无需重启 V2Ray 即可调整配置

## 使用场景

1. **开发调试**：开发者可以方便地测试不同配置
2. **临时切换**：需要频繁切换代理设置的用户
3. **多用户环境**：在局域网中为多个用户提供配置界面
4. **远程管理**：结合隧道技术实现远程管理

## 注意事项

* 此模块主要用于开发和测试，生产环境使用需谨慎
* 浏览器界面不提供身份认证，需自行保证访问安全
* 确保防火墙规则正确配置，避免未授权访问
* 建议仅在本地回环地址上监听，除非有特殊需求
